---
title: Quick Start
description: |
  Start a local MildStack instance, then run first AWS CLI commands for S3, DynamoDB, SQS, and SNS.
---

## 1. Start a Local Instance

```bash
mildstack start
```

<Callout type="info" title="Keep this process running">
  `mildstack start` runs the instance in the current terminal. Keep it open and use another terminal for the commands below.
</Callout>

If MildStack starts on a different port, use that port consistently in the commands below.

## 2. Configure an AWS CLI Profile

```bash
aws configure set aws_access_key_id test --profile mildstack
aws configure set aws_secret_access_key test --profile mildstack
aws configure set region us-east-1 --profile mildstack
```

<Callout type="info" title="Use a dedicated local profile">
  Keep a separate profile (like `mildstack`) so local commands do not mix with your real AWS account profile.
</Callout>

## 3. Run S3 Commands

```bash
aws s3 mb s3://my-bucket \
  --endpoint-url http://localhost:4566 \
  --profile mildstack

aws s3 cp ./hello.txt s3://my-bucket/hello.txt \
  --endpoint-url http://localhost:4566 \
  --profile mildstack

aws s3 ls s3://my-bucket \
  --endpoint-url http://localhost:4566 \
  --profile mildstack
```

## 4. Run DynamoDB Commands

```bash
aws dynamodb create-table \
  --table-name Users \
  --attribute-definitions AttributeName=id,AttributeType=S \
  --key-schema AttributeName=id,KeyType=HASH \
  --billing-mode PAY_PER_REQUEST \
  --endpoint-url http://localhost:4566 \
  --profile mildstack

aws dynamodb put-item \
  --table-name Users \
  --item '{"id": {"S": "user-1"}, "name": {"S": "Alice"}}' \
  --endpoint-url http://localhost:4566 \
  --profile mildstack

aws dynamodb get-item \
  --table-name Users \
  --key '{"id": {"S": "user-1"}}' \
  --endpoint-url http://localhost:4566 \
  --profile mildstack
```

## 5. Run SQS Commands

```bash
QUEUE_URL=$(aws sqs create-queue \
  --queue-name my-queue \
  --endpoint-url http://localhost:4566 \
  --profile mildstack \
  --query 'QueueUrl' \
  --output text)

aws sqs send-message \
  --queue-url "$QUEUE_URL" \
  --message-body "Hello from MildStack!" \
  --endpoint-url http://localhost:4566 \
  --profile mildstack

aws sqs receive-message \
  --queue-url "$QUEUE_URL" \
  --endpoint-url http://localhost:4566 \
  --profile mildstack
```

## 6. Run SNS Commands

```bash
TOPIC_ARN=$(aws sns create-topic \
  --name local-events \
  --endpoint-url http://localhost:4566 \
  --profile mildstack \
  --query 'TopicArn' \
  --output text)

aws sns publish \
  --topic-arn "$TOPIC_ARN" \
  --message "Hello from MildStack SNS" \
  --endpoint-url http://localhost:4566 \
  --profile mildstack

aws sns list-topics \
  --endpoint-url http://localhost:4566 \
  --profile mildstack
```

## 7. Inspect, Stop, and Delete the Instance

Check running instances:

```bash
mildstack instances
```

Then stop and delete by port:

```bash
mildstack stop 4566
mildstack delete 4566
```
