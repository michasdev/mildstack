## Overview

MildStack supports SQS query-style and target-style requests for API version `2012-11-05`.

## Supported Actions

The SQS action catalog currently includes 23 actions, grouped as follows.

### Queue Lifecycle

- `CreateQueue`
- `DeleteQueue`
- `GetQueueUrl`
- `ListQueues`
- `PurgeQueue`
- `GetQueueAttributes`
- `SetQueueAttributes`

### Message Surface

- `SendMessage`, `SendMessageBatch`
- `ReceiveMessage`
- `DeleteMessage`, `DeleteMessageBatch`
- `ChangeMessageVisibility`, `ChangeMessageVisibilityBatch`

### Permissions and Tags

- `AddPermission`, `RemovePermission`
- `TagQueue`, `UntagQueue`, `ListQueueTags`

### DLQ Redrive

- `ListDeadLetterSourceQueues`
- `StartMessageMoveTask`
- `CancelMessageMoveTask`
- `ListMessageMoveTasks`

## Behavior Notes

- If `QueueOwnerAWSAccountId` is provided, it must match the local default account ID.
- Queue delete and purge flows use a cooldown window before repeating the same operation.
- Batch actions enforce the AWS-style maximum of 10 entries per request.
- FIFO message sends require `MessageGroupId`; deduplication follows queue FIFO attributes.

## Queue URL Format

MildStack builds queue URLs with the default local account ID `00000000000` in the path:

```text
http://127.0.0.1:4566/00000000000/<queue-name>
```

## SDK Example

```ts
import {
  SQSClient,
  CreateQueueCommand,
  SendMessageCommand,
} from "@aws-sdk/client-sqs";

const sqs = new SQSClient({
  region: "us-east-1",
  endpoint: "http://127.0.0.1:4566",
  credentials: { accessKeyId: "test", secretAccessKey: "test" },
});

const queueUrl = "http://127.0.0.1:4566/00000000000/orders";
await sqs.send(new CreateQueueCommand({ QueueName: "orders" }));
await sqs.send(new SendMessageCommand({ QueueUrl: queueUrl, MessageBody: "hello" }));
```
