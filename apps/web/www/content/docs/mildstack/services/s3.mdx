## Overview

MildStack currently exposes two S3 surfaces:

- AWS native S3 adapter routes on `/`, `/:bucket`, and `/:bucket/:object`
- Runtime service routes under `/api/v1/runtime/services/s3/...`

They share the same core service implementation but do not currently expose identical operation sets.

## Service Policy and Runtime Routes

The S3 service policy defines the guaranteed operation groups below, and the runtime service routes map those groups to concrete operations.

- `ListBuckets`
- `CreateBucket`
- `HeadBucket`
- `GetBucketLocation`
- `DeleteBucket`

### Bucket Subresources

- Policy: `GetBucketPolicy`, `PutBucketPolicy`, `DeleteBucketPolicy`
- Encryption: `GetBucketEncryption`, `PutBucketEncryption`, `DeleteBucketEncryption`
- Lifecycle: `GetBucketLifecycleConfiguration`, `PutBucketLifecycleConfiguration`, `DeleteBucketLifecycleConfiguration`
- CORS: `GetBucketCors`, `PutBucketCors`, `DeleteBucketCors`
- ACL and tagging: `GetBucketAcl`, `PutBucketAcl`, `GetBucketTagging`, `PutBucketTagging`, `DeleteBucketTagging`
- Ownership and access controls: `GetBucketOwnershipControls`, `PutBucketOwnershipControls`, `DeleteBucketOwnershipControls`, `GetPublicAccessBlock`, `PutPublicAccessBlock`, `DeletePublicAccessBlock`
- Event/admin config: `GetBucketNotificationConfiguration`, `PutBucketNotificationConfiguration`, `GetBucketLogging`, `PutBucketLogging`, `GetBucketReplication`, `PutBucketReplication`, `DeleteBucketReplication`
- Versioning: `GetBucketVersioning`, `PutBucketVersioning`
- Object lock config: `GetObjectLockConfiguration`, `PutObjectLockConfiguration`

### Objects and Multipart

- Object listing and CRUD: `ListObjects` (v1), `ListObjectsV2`, `GetObject`, `HeadObject`, `PutObject`, `CopyObject`, `DeleteObject`, `DeleteObjects`
- Version listing: `ListObjectVersions`
- Object governance: `GetObjectRetention`, `PutObjectRetention`, `GetObjectLegalHold`, `PutObjectLegalHold`, `GetObjectAcl`, `PutObjectAcl`, `GetObjectTagging`, `PutObjectTagging`, `DeleteObjectTagging`
- Multipart: `CreateMultipartUpload`, `UploadPart`, `ListParts`, `CompleteMultipartUpload`, `AbortMultipartUpload`, `ListMultipartUploads`

## Current Native Adapter Coverage

The current AWS native S3 adapter routes (used by AWS SDK and CLI on the local endpoint) handle these operations today:

- Buckets: `ListBuckets`, `CreateBucket`, `HeadBucket`, `DeleteBucket`
- Bucket subresources: policy, encryption, lifecycle, CORS, ACL, tagging, ownership controls, public access block, versioning
- Objects: `ListObjects` (v1), `ListObjectsV2`, `GetObject`, `HeadObject`, `PutObject`, `CopyObject`, `DeleteObject`, `DeleteObjects`
- Multipart: `CreateMultipartUpload`, `UploadPart`, `CompleteMultipartUpload`

<Callout type="warn" title="Native adapter scope today">
  Some policy-covered operations are currently exposed only through runtime service routes, not through the AWS native S3 adapter. Examples include bucket location, notification/logging/replication, object lock and governance subresources, and multipart listing or abort actions.
</Callout>

## Deferred Scope

The current S3 policy explicitly defers these areas:

- Directory buckets / S3 Express, including `CreateSession`, `ListDirectoryBuckets`, `RenameObject`, and related metadata configuration flows
- Reporting and admin surfaces such as analytics, intelligent-tiering, inventory, and metrics
- Lower-value management features such as website hosting, accelerate, request payment, ABAC, and policy status
- Specialized data-plane / Object Lambda actions such as `GetObjectAttributes`, `GetObjectTorrent`, `RestoreObject`, `SelectObjectContent`, `UpdateObjectEncryption`, `UploadPartCopy`, and `WriteGetObjectResponse`

## SDK Example

```ts
import { S3Client, CreateBucketCommand, PutObjectCommand } from "@aws-sdk/client-s3";

const s3 = new S3Client({
  region: "us-east-1",
  endpoint: "http://127.0.0.1:4566",
  credentials: { accessKeyId: "test", secretAccessKey: "test" },
  forcePathStyle: true,
});

await s3.send(new CreateBucketCommand({ Bucket: "photos" }));
await s3.send(new PutObjectCommand({ Bucket: "photos", Key: "hello.txt", Body: "hello" }));
```

<Callout type="info">
  Use path-style addressing (`forcePathStyle: true`) with the AWS SDK for JavaScript.
</Callout>
