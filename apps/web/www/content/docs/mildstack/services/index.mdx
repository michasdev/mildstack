## Service Coverage

MildStack currently runs four AWS-compatible services in the core runtime:
S3, DynamoDB, SQS, and SNS.

<Cards>
  <Card title="Amazon S3" href="/docs/mildstack/services/s3">

  </Card>
  <Card title="Amazon DynamoDB" href="/docs/mildstack/services/dynamodb">

  </Card>
  <Card title="Amazon SQS" href="/docs/mildstack/services/sqs">

  </Card>
  <Card title="Amazon SNS" href="/docs/mildstack/services/sns">

  </Card>
</Cards>

## Compatibility Model

Compatibility is documented by operation, not by blanket parity claims.

- Each service page separates guaranteed policy coverage from adapter-specific behavior when they differ.
- See [Compatibility & Limits](/docs/mildstack/services/compatibility-limits) for a cross-service matrix.

<Callout type="warn" title="S3 scope note">
  The S3 service policy and runtime service routes cover more operations than
  the current AWS native S3 adapter. See the S3 page for the exact split.
</Callout>

## Local Connection Defaults

| Setting | Value |
| --- | --- |
| Endpoint | `http://127.0.0.1:4566` (default) |
| Region | `us-east-1` |
| Default account ID | `00000000000` |
| Access key / secret key | Any placeholder values |

## Quick Start Example

```ts
import { SQSClient } from "@aws-sdk/client-sqs";

const sqs = new SQSClient({
  region: "us-east-1",
  endpoint: "http://127.0.0.1:4566",
  credentials: {
    accessKeyId: "test",
    secretAccessKey: "test",
  },
});
```

MildStack does not require real AWS credentials for local development.
