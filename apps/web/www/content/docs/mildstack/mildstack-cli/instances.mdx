---
title: Instances
description: |
  Manage multiple isolated MildStack instances and their lifecycle from the CLI.
---

An instance is one local MildStack runtime bound to one port.
Each instance has isolated service data through its own `instanceId`.

## Start and Detach

Use `start` to launch a runtime on a port. `--detach` and `-d` are equivalent, and the flag can appear before or after the port.

```bash
mildstack start 4566
mildstack start 4566 --detach
mildstack start --detach 4566
```

If no port is provided, `start` probes ports `4566`, `4567`, and `4568`.

## Inspect Runtime State

Use `instances` for the human-readable snapshot and `instances --json` when you need machine-readable output. `status` is an alias of `instances` and returns the same data.

```bash
mildstack instances
mildstack instances --json
```

## Lifecycle States

Instances go through these states:

| State | Description |
| --- | --- |
| **running** | The instance is active and accepting requests |
| **not_started** | The instance exists on disk but is not currently running |
| **errored** | The instance failed to start; error details are included in output |

## Stop

`stop` targets running instances only.

```bash
mildstack stop 4566
mildstack stop --all
```

## Delete

`delete` removes instance records and instance resources from disk.

```bash
mildstack delete 4566
mildstack delete --all
```

<Callout type="warn" title="Required target selection">
  `stop` and `delete` require a specific port or `--all`.
  `--all` cannot be combined with a port argument.
</Callout>

## Data Isolation

- Each instance is isolated by `instanceId` and port.
- Stopping preserves instance data.
- Deleting removes instance data for that instance.

Storage root: `~/.mildstack/instances/`
