MildStack exposes local AWS-compatible endpoints.
Point AWS CLI commands to your running instance (`http://localhost:<port>`).

## Create a Dedicated CLI Profile

```bash
aws configure set aws_access_key_id test --profile mildstack
aws configure set aws_secret_access_key test --profile mildstack
aws configure set region us-east-1 --profile mildstack
```

<Callout type="info">
  MildStack does not require real AWS credentials. Any placeholder values work as long as the AWS CLI has credentials configured.
</Callout>

## Start an Instance

```bash
mildstack start
```

By default, MildStack starts on `4566`. If you use a different port, set `MILDSTACK_PORT` before running AWS CLI commands.

## Set Endpoint and Profile

Set the port once and reuse it in a small wrapper:

```bash
export MILDSTACK_PORT=4566

awslocal() {
  aws --endpoint-url "http://localhost:${MILDSTACK_PORT}" --profile mildstack --region us-east-1 "$@"
}
```

<Callout type="idea" title="Multiple local instances">
  Change `MILDSTACK_PORT` when you switch instances. For a one-off command, you can override it inline, for example `MILDSTACK_PORT=8080 awslocal s3 ls`.
</Callout>

## Service Examples

### S3

```bash
awslocal s3 ls
awslocal s3 mb s3://my-bucket
awslocal s3 cp ./data.json s3://my-bucket/data.json
awslocal s3 ls s3://my-bucket
awslocal s3 rm s3://my-bucket/data.json
awslocal s3 rb s3://my-bucket
```

### DynamoDB

```bash
awslocal dynamodb create-table \
  --table-name Users \
  --attribute-definitions AttributeName=id,AttributeType=S \
  --key-schema AttributeName=id,KeyType=HASH \
  --billing-mode PAY_PER_REQUEST

awslocal dynamodb list-tables

awslocal dynamodb put-item \
  --table-name Users \
  --item '{"id": {"S": "user-1"}, "name": {"S": "Alice"}, "email": {"S": "alice@example.com"}}'

awslocal dynamodb get-item \
  --table-name Users \
  --key '{"id": {"S": "user-1"}}'

awslocal dynamodb delete-table --table-name Users
```

### SQS

```bash
QUEUE_URL=$(awslocal sqs create-queue --queue-name my-queue --query 'QueueUrl' --output text)

awslocal sqs list-queues

awslocal sqs send-message \
  --queue-url "$QUEUE_URL" \
  --message-body '{"event": "order.created", "orderId": "123"}'

awslocal sqs receive-message \
  --queue-url "$QUEUE_URL"

awslocal sqs delete-queue \
  --queue-url "$QUEUE_URL"
```

### SNS

```bash
TOPIC_ARN=$(awslocal sns create-topic --name local-events --query 'TopicArn' --output text)
awslocal sns list-topics
awslocal sns publish --topic-arn "$TOPIC_ARN" --message 'hello from MildStack'
```

## Notes

- The `awslocal` wrapper reads `MILDSTACK_PORT`, so you only need to change the port value when you switch instances.
- Use `--output json` on AWS CLI commands when scripting.
